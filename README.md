# JobFinder FastAPI

## Что реализовано в этом обновлении

### 1) Полноценный блок администратора (`/api/v1/admin`)

Администратор получил набор функций, характерный для job-платформы (в стиле HH):

- **Управление справочниками** (полный CRUD):
  - `cities`
  - `professions`
  - `skills`
  - `currencies`
  - `experiences`
  - `statuses`
  - `work-schedules`
  - `employment-types`
  - `educational-institutions`

- **Модерация пользователей**:
  - список пользователей с фильтрами (`role`, `is_active`, `search`)
  - блокировка/разблокировка аккаунтов через `PATCH /users/{user_id}/status`

- **Модерация компаний и соискателей**:
  - просмотр компаний и удаление компании
  - просмотр соискателей и удаление профиля соискателя

- **Модерация вакансий**:
  - список вакансий с фильтрами (`search`, `status_id`, `city_id`, `profession_id`, `company_id`)
  - просмотр вакансии
  - смена статуса вакансии (архив/публикация/пауза и т.д. через `status_id`)
  - удаление вакансии

- **Контроль откликов**:
  - просмотр откликов в системе с фильтром по статусу

Все admin-endpoint’ы защищены проверкой роли `admin`.

---

### 2) Публичный блок для незарегистрированных пользователей (`/api/v1/public`)

Реализованы endpoint’ы, которые не требуют авторизации и дают только просмотр активных вакансий:

- `GET /public/vacancies`
  - лента активных вакансий
  - поддержка фильтров: `city_id`, `profession_id`, `company_id`, `search`
  - пагинация: `skip`, `limit`

- `GET /public/vacancies/{vacancy_id}`
  - карточка активной вакансии
  - расширенные поля для фронтенда: компания, город, профессия, график, тип занятости, валюта, опыт, навыки

Отклик без авторизации не поддерживается (и не реализован).

---

## Готовность под фронтенд

При проектировании API добавлены:

- единый префикс и раздельные роутеры по доменам;
- фильтры + пагинация для таблиц и списков;
- response-модели, пригодные для админ-панелей и публичной витрины;
- предсказуемые 404/409/403 сценарии для UI-ошибок.
